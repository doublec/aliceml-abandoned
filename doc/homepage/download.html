Alice is based on the upcoming version 1.2.5 of the Mozart Programming system, which has to be installed before you can install Alice itself. Previous Mozart versions cannot run Alice! We provide snapshot packages of Mozart 1.2.5 below until it has been officially released.

All packages include the full manual web pages.

The current version of the Alice system is somewhat resource-hungry. We hence recommend a system with at least 700 MHz and 256 MB Ram. The installation requires approximately 30 MB of Harddisk space (including Mozart).

  • mozart-1.2.5-20030124.exe
  • alice-0.9.1-20030126.exe (Alice 0.9.1 Bugfix Release Candidate (try first))
  • alice-0.9-20021220.exe (Alice 0.9 Release (try if 0.9.1 fails))

If you use non-standard installation paths for either package, you have to set the following environment variables appropriately (depending on your shell - the following example uses CMD.EXE or COMMAND.COM syntax): 

set OZHOME=<your mozart installation directory>
set STOCKHOME=<your alice installation directory>

If you have trouble installing Mozart, please consult the respective installation notes for Windows.

Red Hat 7.2:

  • Mozart 1.2.5 is not yet available as an RPM, please use the tarball below.
  • alice-0.9-1.noarch.rpm

Debian testing (sarge)

  • mozart_1.2.5.20030120-1_i386.deb
  • mozart-contrib_1.2.5.20030120-1_i386.deb
  • alice_0.9.20030120-1_i386.deb

(You need to install all three packages to make full use of Alice.)

Other Linuxes (binary tarballs):

  • mozart-1.2.5-20021218.tgz
  • alice-0.9-20021220.tgz

If you use either tarball, you have to include the respective bin directory into your PATH, e.g. 

PATH=<mozart-directory>/bin:<alice-directory>/bin:$PATH

In order to use the interactive Alice top-level from within Emacs, the following set-up is necessary.

First make sure that the environment variables OZHOME and STOCKHOME point to the installation directory of the Mozart system and the Alice system, respectively.

To install the patched SML mode version 3.9.5, extract the mode archive file and follow the installation instructions given in INSTALL. Then invoke load-library with argument sml-proc. Afterwards, invoke customize-group with argument sml and adjust the settings sml-program-name to alice.

In addition to the released version, the current development sources are also available via anonymous CVS.

Warning: It is quite likely that development snapshots might have bugs, they might not even build! You are basically on your own.

CVSROOT:  :pserver:anoncvs@cvs.ps.uni-sb.de:/services/alice/CVS
Module:   alice
Password: anoncvs

The tag for the current release branch is:

operette3

An example session to check out the whole Alice source tree (including tools, libraries, and documentation) might look as follows:

% cvs -d :pserver:anoncvs@cvs.ps.uni-sb.de:/services/alice/CVS login
(Logging in to anoncvs@cvs.ps.uni-sb.de)
CVS password: anoncvs
% cvs -d :pserver:anoncvs@cvs.ps.uni-sb.de:/services/alice/CVS get alice

From the CVS repository you can also access the released version:

% cvs -d :pserver:anoncvs@cvs.ps.uni-sb.de:/services/alice/CVS get -r operette3 alice

You can likewise access the Mozart CVS repository.
